# Product Definition

## One-Line Pitch

VitalClaw is a personal health observability system that learns your normal, catches meaningful drift, and remembers what happened after you acted.

## What The Product Is

- A longitudinal monitoring engine for one person.
- A memory layer for health episodes, context, interventions, and outcomes.
- An explanation surface that can be used through Codex today and other interfaces later.

## What The Product Is Not

- Not a generic AI health chatbot.
- Not a prettier Apple Health dashboard.
- Not a diagnosis engine.
- Not a clinic workflow product in v1.

## First User

The first target user is an Apple Watch power user who already checks sleep, recovery, and body-state signals, but wants a better answer to:

`"Is this just a bad day, or is something genuinely drifting?"`

## Core Promise

VitalClaw should help the user say:

`"It caught a change I would not have noticed early enough on my own."`

## Differentiation

VitalClaw should be strongest in four places:

- context-aware personal baselines
- episode memory
- low-noise alert policy
- action and follow-up loops

Those are the product. The chat interface is not.
